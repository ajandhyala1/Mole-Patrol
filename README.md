This project was built using an STM32F401RE Nucleo board, two breadboards, five push buttons, five LEDs, and a 16x2 LCD display operating in 4-bit mode. The game logic was written entirely in ARM assembly language and uses GPIO interfacing for all user inputs and outputs. The system was physically divided into two sections: a top breadboard containing the gameplay interface (LEDs and buttons) and a bottom breadboard containing the LCD scoreboard and contrast potentiometer.

The LED system consists of five LEDs connected to STM32 GPIO output pins. The LEDs are mapped as follows: LED1 → PA0, LED2 → PA1, LED3 → PA4, LED4 → PB0, and LED5 → PC1. Each LED should be connected in series with a current-limiting resistor, typically around 220Ω, before connecting to ground. Although one LED was removed during physical testing after hardware failure, the intended design and software fully support all five LEDs.

The button input system uses five push buttons connected to GPIO input pins on the STM32. The buttons are mapped as follows: Button 1 → PC0, Button 2 → PC2, Button 3 → PC3, Button 4 → PB3, and Button 5 → PB4. Each button can be wired using a pull-down resistor configuration where pressing the button drives the corresponding input pin high. Button 1 and Button 2 are additionally used to select the game difficulty, with Button 1 selecting Easy Mode and Button 2 selecting Hard Mode.

The LCD scoreboard is implemented using a standard 16x2 character LCD configured in 4-bit communication mode to reduce GPIO usage. The LCD pin mapping is as follows: RS → PB5, EN → PA6, D4 → PA7, D5 → PA8, D6 → PA9, and D7 → PA10. Additional LCD connections include RW tied to ground, VSS connected to ground, VDD connected to +5V, and VO connected to the output of a potentiometer used for display contrast adjustment.

The STM32F401RE microcontroller executes all game functionality directly in ARM assembly language without external libraries. The program generates pseudo-random LED activation patterns to simulate “moles,” reads button presses from the player, tracks the current score, manages timing and difficulty modes, and continuously updates the LCD scoreboard in real time. The project supports both Easy and Hard gameplay modes, pseudo-random mole generation, LCD score output, and button-based user interaction.

This project was developed collaboratively by Aditya Jandhyala and Dalton Moore for ESET 349: Microcontroller Architecture as a final embedded systems project focused on low-level programming, GPIO interfacing, hardware/software integration, and microcontroller-based system design.
